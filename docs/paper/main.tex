\begin{abstract}
Coding agents overwhelmingly edit source code as text, and small locally
runnable models fail at it: diff formats break, identifiers are guessed,
and errors surface only after files change. We present \textbf{ago}, a
semantic edit protocol for Go in which an agent queries a typechecked
workspace and submits mutations from a fixed operation catalog; every
mutation is validated by the compiler against an in-memory snapshot
before any file is written, applied as an atomic generation-checked
transaction, or rejected. Rejections are designed as conversation:
each carries typed diagnostics and \emph{paste-ready repairs} --- the
corrected call, verbatim --- and exact resends escalate. We introduce an
\emph{oracle} methodology that replays ground-truth commits through the
protocol to certify every benchmark task solvable before any model time
is spent, and mine tasks from real commit histories of traefik, vault,
boundary, and cobra. Across \EpisodeCount{} recorded episodes with a
quantized 30B-class local model, the semantic mode passes
\SemanticPassRate{} of episodes versus \RawPassRate{} for raw file
editing under identical prompts and budgets, with lower median
time-to-green on nearly every task; a 9B-class model completes
repository-scale refactoring tasks through the protocol on first
contact. All code, per-episode evidence (transcripts, request logs,
serving configurations), and analysis tooling are released.
\end{abstract}

\section{The ago Protocol}
\subsection{Surface}
Ten tools; a versioned catalog of 28 operations across declaration,
statement, and test families; seven query kinds with deterministic,
paged responses. Handles address statements inside declarations;
generations make staleness explicit.
\subsection{Transactions}
An ordered op list validates as one unit against an in-memory snapshot:
ops apply to a copy, the dirty set re-typechecks once (in parallel,
scheduled over post-edit import edges), resolution proofs run (rename
rejects reference capture the compiler would accept), then all files
write together --- or nothing does. Pre-existing diagnostics are
baselined and tolerated; only new ones reject.
\subsection{Rejection as conversation}
Every rejection answers ``what is the correct next call'': typed
diagnostics, candidate lists, and \texttt{possible\_repairs} --- complete
calls the agent resends verbatim. Repairs are tested by execution: a
repair that would itself reject is never offered. Exact resends of a
rejected call escalate. Accepted mutations embed the fresh view of the
touched declaration, closing the edit-view loop without a round trip.

\section{Oracle Methodology}
Ground-truth changes mined from real commits are replayed through the
protocol itself. A task enters the benchmark only when the oracle
completes it; oracle rejections are findings (engine bugs, extractor
errors, or named protocol ceilings), and oracle transcripts are a
training corpus. This separates ``the model failed'' from ``the task was
impossible'' by construction.

\section{Benchmark}
Same model, same harness, same prompts (token-parity enforced by test),
same time cap; one arm edits files with shell tools, the other sees only
the protocol. Scoring is goal predicate + typecheck + scoped tests, the
test gate voided where a pristine worktree also fails it. Serving
configurations are pinned per run; every episode records transcript,
diff, score, token usage, failure kind, and the daemon's request log.
