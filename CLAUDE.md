## Beads Issue Tracker

This project uses **bd (beads)** for issue tracking. Run `bd prime` to see full workflow context and commands.

### Quick Reference

```bash
bd ready              # Find available work
bd show <id>          # View issue details
bd update <id> --claim  # Claim work

```

### Rules

- Use `bd` for ALL task tracking — do NOT use TodoWrite, TaskCreate, or markdown TODO lists
- Run `bd prime` for detailed command reference and session close protocol
- Use `bd remember` for persistent knowledge — do NOT use MEMORY.md files

## Session Completion

**When ending a work session**, you MUST complete ALL steps below. Work is NOT complete until `git push` succeeds.

**MANDATORY WORKFLOW:**

1. **File issues for remaining work** - Create issues for anything that needs follow-up
2. **Run quality gates** (if code changed) - Tests, linters, builds
3. **Update issue status** - Close finished work, update in-progress items
4. **PUSH TO REMOTE** - This is MANDATORY:
   ```bash
   git pull --rebase
   bd dolt push
   git push
   git status  # MUST show "up to date with origin"
   ```
5. **Clean up** - Clear stashes, prune remote branches
6. **Verify** - All changes committed AND pushed
7. **Hand off** - Provide context for next session

**CRITICAL RULES:**
- Work is NOT complete until `git push` succeeds
- NEVER stop before pushing - that leaves work stranded locally
- NEVER say "ready to push when you are" - YOU must push
- If push fails, resolve and retry until it succeeds

## Build & Test

```bash
go test ./...                              # full suite; snapshot tests ~90s, bench model runs self-skip
gofmt -l .                                 # must be empty
go test -race ./internal/snapshot          # required for retypecheck/concurrency changes (also in CI)

# bench
AGO_BENCH_MODES=oracle AGO_BENCH_SCRATCH=~/.cache/ago-bench/clones \
  go test ./bench -run OracleSweep -parallel 4 -timeout 0    # certify ground truth, no model
AGO_BENCH_PROFILE=<name> AGO_BENCH_SCRATCH=~/.cache/ago-bench/clones \
  go test ./bench -bench Rename -benchtime 3x -timeout 0     # model round (profiles.json)
# AGO_BENCH_SUITE=smoke: one certified task per kind, smallest repo (first contact tier)
go run ./cmd/bench certify bench/results/<run> ...           # flip certified flags from oracle evidence
go run ./cmd/bench report|export|paper bench/results/<run>   # Wilson CIs | MLflow | LaTeX tables
```

CI (`.github/workflows/ci.yml`) runs gofmt, vet, the full suite, and race on every push.

## Architecture Overview

`ago` is a semantic edit protocol for Go: agents query a typechecked
workspace and submit compiler-validated mutations, never raw text edits.
One binary, per-workspace daemon (auto-spawn, unix socket, idle exit).

- `internal/snapshot/`: the engine. Typechecked snapshot, parallel
  retypecheck, op implementations (`ops_*.go`), composable patch pipeline
  (`patch.go`), repair engine (`repairs.go`), help catalog (`help.go`).
- `internal/daemon/`: socket server, request log, resend breaker.
- `cmd/ago/`: CLI verbs + MCP server (`ago mcp`) + `ago init` scaffold.
- `bench/`: raw-vs-semantic comparison harness; tasks mined from real
  commits (traefik, vault, boundary, cobra); the oracle replays ground
  truth to certify tasks before any model time.
- `cmd/bench/`: mining, extraction, certify, report, MLflow export,
  paper table generation.

Doc map: `docs/specs/language.md` (op catalog and guarantees, canonical),
`docs/specs/surface.md` (shipped/planned calls), `docs/specs/protocol.md`
(semantics), `docs/specs/bench.md`, `docs/tenets.md` (engineering
principles, each with its named enforcement), `idea.md` (thesis),
`docs/adr/` (append-only decisions), `docs/optimizations/` (per-model
serving research), `docs/paper/` (arXiv draft; tables generated by
`bench paper`, never hand-edited).

## Conventions & Patterns

- Strict TDD: failing test first, watch it fail, then implement. Every
  op and repair landed that way.
- The guard lattice enforces doc/code sync; when a guard fails it names
  the file to fix. Adding an op means: registry + help catalog entry +
  `catalogVersion` bump + recorded hash in `guards_test.go` + surface.md
  row; the guards walk you through it.
- `internal/snapshot/testdata/demo/lib` is frozen behind a recorded hash
  (`TestDemoLibFixtureFrozen`); new fixture shapes go in `demo/sig`.
- Model rounds run only oracle-certified tasks. Certification comes from
  `bench certify` over committed evidence, never by hand-editing task
  JSON.
- Bench evidence in `bench/results/` is append-only; never rewrite or
  regenerate committed runs. Every episode must carry task, mode, and
  profile identity.
- No sidecar scripts (`TestNoSidecarScripts`): a script is a missing Go
  subcommand.
- Repairs are executed verbatim in tests before being offered; a new
  repair kind lands with its paste-back execution test.
- No em or en dashes anywhere: commits, docs, code comments. Terse,
  verb-first commit messages, no attribution footers.
- ADRs are append-only; supersede, never rewrite.
- Ceilings are named where they bind ("v1 ceiling: ..., rejects with the
  blocker named"); a silent wrong answer is the bug.
